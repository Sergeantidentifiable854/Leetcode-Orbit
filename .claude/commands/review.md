## 第三步：P0/P1/P2 分析

### P0 — 正确性
- 语法是否正确（能否通过 Python 解释器）
- 是否通过所有测试用例（记录通过率 x/y）
- 是否有 RuntimeError（除零、越界、空指针）
- 输出格式是否正确

### P1 — 算法与复杂度
- 算法选择是否合理（与参考答案对比）
- 时间复杂度分析（标注用户的 vs 最优的）
- 空间复杂度分析
- 是否有不必要的重复计算

### P2 — 代码写法
- Pythonic 风格（enumerate、zip、列表推导等）
- 变量命名是否清晰
- 是否有冗余代码
- 边界条件处理是否优雅

### 薄弱点识别

薄弱点有两个来源，**必须合并去重**：

**来源 1 — 提示过程中的薄弱点：**
读取 `workspace/.current_meta.json` 中的 `hint_weaknesses` 字段（如果存在）。这些薄弱点是在 `/hint` 阶段根据用户求助内容判断的，反映了用户"不会做"的部分。

**来源 2 — 代码分析中的薄弱点：**
根据最终代码的分析结果，识别薄弱点标签（简短关键词），分类为：
- 算法思路类: "未识别最优算法", "逻辑错误"
- 数据结构类: "哈希表不熟练", "链表操作有误"
- 边界条件类: "空输入未处理", "单元素未处理"
- 编码能力类: "不够Pythonic", "命名不规范"
- 复杂度类: "时间复杂度未达最优", "空间可优化"

**合并规则：** 将来源 1 和来源 2 的薄弱点合并为一个列表，去除重复项。即使最终代码全部通过，只要 `hint_weaknesses` 中有内容，也算"有薄弱点"。

## 第四步半：知识讲解 + 相关题推荐

### 知识讲解

根据用户代码与最优解的差距，**有针对性地讲解相关概念**：

- **用户用了非最优算法**（如暴力枚举）：
  - 简要点明存在更优的方法和核心数据结构/算法名称
  - 不需要展开讲原理（用户可能下次自己想出来），但要给出关键思路
  - 例："这题用哈希表可以做到 O(n)——遍历时把见过的数存进字典，每次检查 target-num 是否已存在"

- **用户用了正确算法但实现有问题**（如用了哈希表但写法有误）：
  - 重点讲解该算法/数据结构的原理和关键细节
  - 指出用户具体哪里理解有偏差
  - 例："哈希表查找是 O(1) 的，但你在存入时机上有问题——应该先查再存，否则同一个元素会和自己配对"

- **用户完美通过**：
  - 简要总结该题涉及的核心思想
  - 可以提一个进阶变体让用户思考（如"如果要返回所有配对呢？"）

格式：
```
📖 知识点:
{2-5句讲解，聚焦用户最需要补的部分}
```

1. 同标签且 `pw` 权重高的（薄弱的优先）
2. 同标签且 `att == 0` 的（没做过的优先）
3. 同分类但不同标签的（拓展视野）

格式：

```
  - LeetCode {id}. {title} ({difficulty}) [{tags}]
```

### 路径B: 更新 data/hot100_index.json

更新当前题的字段：
- `att`: += 1
- `cor`: 如果全部测试通过则 += 1

### 路径D: 追加 hot100_user_analysis.md

如果文件不存在则创建（含表头）。

**每次 review 后都要重新统计并更新文件顶部的进度区域。** 从 `data/hot100_index.json` 中统计：

- **总进度**: `att > 0` 的题目数 / 100
- **熟练**: `cor >= 2` 且 `weak` 为空 且 `pw < 0.5` 的题目数（多次通过、无薄弱点、权重已降低）
- **不熟练**: `att > 0` 且 (`cor == 0` 或 `weak` 不为空 或 `pw >= 1.5`) 的题目数（做过但没掌握）
- **一般**: 已做但不属于以上两类的题目数（做过、通过过、但还需巩固）
- **未刷**: `att == 0` 的题目数

文件顶部格式：

```markdown

> 连续打卡: {N} 天 | 本周刷题: {N} 题

## 📈 题库进度

总进度: [████████░░░░░░░░░░░░] {done}/100 ({done}%)

✅ 熟练 ({N}题):  [██████░░░░] {N}%  多次通过、无薄弱点
⚠️ 一般 ({N}题):  [████░░░░░░] {N}%  通过过、仍需巩固
❌ 不熟练({N}题): [██░░░░░░░░] {N}%  做过但未掌握
⬜ 未刷 ({N}题):  [████████░░] {N}%  尚未尝试
```
